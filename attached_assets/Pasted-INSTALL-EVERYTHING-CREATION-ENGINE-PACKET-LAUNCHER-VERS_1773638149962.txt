CORE PLATFORM BEHAVIOR:
- The platform is a CLEAN LAUNCHER, not a container.
- The system NEVER opens projects inside itself.
- Every project is generated as an EXTERNAL PACKET (file, folder, or standalone page).
- The system only displays:
  - Project Title
  - Project Description (optional)
  - “Open Packet” button/link
- Clicking the link opens the packet OUTSIDE the system (new tab or file viewer).
- The platform stays simple, clean, uncluttered, and never overwhelming.

PACKET SYSTEM:
- Every project (Healthcare, Legal, Construction, Games, Movies, Business, Creative, etc.) becomes a PACKET.
- A packet is a standalone mini‑project containing:
  - structure
  - layout
  - templates
  - visuals
  - content
  - flows
  - multimedia placeholders
  - project‑specific logic
  - its own AI editor
- Packets are stored by the system but NOT displayed inside it.
- Packets open externally, separate from the platform UI.

PROJECT‑SPECIFIC INTELLIGENCE:

  - layouts
  - templates
  - components
  - visuals
  - structures
  - flows
- It NEVER overwhelms the user with unnecessary options.
- It chooses smart defaults that fit the project category.
- It keeps everything simple, clean, and easy to understand.

PACKET‑SPECIFIC AI EDITOR:
