1) Rewrite artifacts/api-server/src/routes/habits.ts

Goals:
- Add guardian approval for kids’ habits
- Use outboundEngine for in-app notifications
- Keep the file clean and self-contained

Requirements:

Imports:
- Keep existing imports (express, auth, db, etc.)
- Ensure outboundEngine and getSql are imported

Endpoints:

A. GET /api/habits
- Return all habits for the current user/family
- For each habit, include:
  - pending_approvals: number of pending completions for that habit
  - has_pending_today: boolean, true if there is a pending completion for this habit created today
- total/streak logic must only count completions where approval_status = 'approved'

B. POST /api/habits/:id/complete
- Look up the current user’s family role (e.g., family_child, family_adult, founder)
- If role = 'family_child':
  - Insert a completion row with:
    - approval_status = 'pending'
    - reviewed_at = NULL
    - reviewed_by = NULL
  - Do NOT update streaks
  - Use outboundEngine to send in-app notifications to all adults:
    - channel: "in-app"
    - title: "Habit completion pending review"
    - body: "<childName> completed <habitName>. Review required."
- If role is adult/founder:
  - Insert completion with approval_status = 'approved'
  - Recalculate and update streaks as currently implemented
- All streak calculations must filter by approval_status = 'approved'

C. GET /api/habits/approvals
- Auth required
- Only allow adults/founder roles; return 403 for children
- Return a list of pending completions:
  - completion_id
  - habit_id
  - habit_name
  - habit_emoji (if available)
  - child_name
  - created_at (submission time)

D. POST /api/habits/approvals/:completionId/approve
- Auth required, adults/founder only
- Set:
  - approval_status = 'approved'
  - reviewed_at = NOW()
  - reviewed_by = current user id
- Recalculate streaks for that habit and update as needed
- Notify the child via outboundEngine:
  - channel: "in-app"
  - title: "Habit approved"
  - body: "Your habit '<habitName>' was approved!"

E. POST /api/habits/approvals/:completionId/reject
- Auth required, adults/founder only
- Set:
  - approval_status = 'rejected'
  - reviewed_at = NOW()
  - reviewed_by = current user id
- Notify the child via outboundEngine:
  - channel: "in-app"
  - title: "Habit needs another try"
  - body: "Your habit '<habitName>' needs another try."

General rules:
- Children can create completions but never approve/reject.
- Only approved completions count toward streaks, rewards, and any bank logic.
- Pending and rejected completions do not affect streaks.
