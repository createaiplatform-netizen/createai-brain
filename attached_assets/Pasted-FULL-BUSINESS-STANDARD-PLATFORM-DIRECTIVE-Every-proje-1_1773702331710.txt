FULL BUSINESS STANDARD — PLATFORM DIRECTIVE
===========================================

Every project generated by this platform must be structured and presented as a
REAL, COMPLETE, PROFESSIONAL business system — not a demo, not a mockup, not a
placeholder.

This means:

1. BUSINESS-GRADE COMPLETENESS
------------------------------
Each project must include:
- clear workflows
- structured processes
- defined roles and responsibilities
- realistic data models
- functional screens and components
- meaningful interactions
- end-to-end flow from overview → detail → action

Nothing should appear empty, fake, or unfinished.

2. EXPERT-LEVEL QUALITY
-----------------------
Every project must be designed as if:
- industry experts
- compliance specialists
- operations leaders
- workflow designers
- UI/UX professionals

all contributed to it.

This means:
- organized structure
- clear logic
- professional tone
- consistent patterns
- realistic business artifacts (forms, checklists, workflows, dashboards)

3. INDUSTRY-READY DEPTH
-----------------------
Each project must feel like a real business in its industry.

Examples:
- Healthcare projects must reflect healthcare workflows, safety, compliance, and operations.
- Construction projects must reflect tools, tasks, safety checks, scheduling, and materials.
- Staffing projects must reflect sourcing, onboarding, credentialing, scheduling, and QA.

Every industry must feel supported with real, meaningful structure.

4. FUNCTIONALITY EXPECTATION
----------------------------
All visible elements must:
- work
- respond
- update
- show detail
- show depth
- show logic

No dead buttons.  
No empty pages.  
No “coming soon.”

5. DEMO → TEST → BUY FLOW
--------------------------
Every project must support:
- a polished demo experience
- a deeper test experience with richer data
- a clear path to “purchase” or “activate” the project

This means:
- demo mode shows the concept clearly
- test mode shows realistic workflows and data
- the project feels complete enough to evaluate seriously

6. UNIVERSAL EFFICIENCY PRINCIPLES
----------------------------------
Every project must follow:
- safety-first logic
- compliance-first logic
- waste reduction
- fraud prevention
- no duplicate data entry
- automation wherever possible
- clarity and simplicity for the user

7. CONSISTENT EXPERIENCE
------------------------
All projects must feel like they belong to the same platform:
- same layout patterns
- same navigation logic
- same depth model
- same quality standard

8. REAL BUSINESS INTENT
-----------------------
Every project must be built with the assumption that:
- a real company might use it
- a real team might run operations through it
- a real buyer might evaluate it
- a real workflow might depend on it

This is not a toy.  
This is not a sample.  
This is a platform for real businesses.

FINAL RULE
----------
If a project is generated, it must be:
- complete
- functional
- structured
- meaningful
- safe
- compliant
- efficient
- professional

It must look and feel like a real business system built by experts.
